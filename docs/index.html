01 Premise

BusyBox-style multi-call binary, hand-written in NASM.

The multi-call family already had three flavours: jib in Rust, topsail in Go, mainsail in Python. Rill is the same shape rebuilt at the lowest practical level — every applet hand-written in NASM, every operation a direct syscall, no dynamic linker, no libc, no global allocator, no startup glue.

It's not trying to be smaller than busybox. It's trying to be readable end-to-end: every byte the kernel sees originated in a .asm file in this repo.

02 Specs

What's in the box.

BINARY
One ~108 KB static ELF, two PT_LOAD segments, ldd says "not dynamic".
APPLETS
41 POSIX utilities — file ops, text, paths, system, control. Each implements common flags.
RUNTIME
No libc · no startup runtime · no global allocator. .bss + stack frames only.
DISPATCH
Multi-call via argv[0] basename, or rill <applet> subcommand.
TESTS
329 bats cases — byte-diff against /usr/bin/<applet> where possible.
STACK
NASM 2.x · GNU ld · GNU make · System V AMD64 ABI · Linux x86_64.
03 Quickstart

Download, link, dispatch.

From a release no toolchain is required. From source: NASM, GNU ld, GNU make, and bats — Linux or WSL.

# From a release — no toolchain needed (Linux x86_64)
$ curl -LO https://github.com/Real-Fruit-Snacks/rill/releases/latest/download/rill-linux-x64
$ chmod +x rill-linux-x64 && mv rill-linux-x64 rill
$ ./rill date
$ ln -s rill ls && ./ls -la              # multi-call via symlink

# From source — NASM + ld + make + bats
$ git clone https://github.com/Real-Fruit-Snacks/rill.git && cd rill
$ make             # builds build/rill
$ make symlinks    # build/<applet> symlinks for multi-call dispatch
$ make test        # 329-case bats integration suite
$ make size        # prints the linked binary's size

# Verify
$ file build/rill                         # ELF 64-bit, statically linked
$ ldd  build/rill                         # not a dynamic executable
$ ./build/rill true && echo ok            # subcommand dispatch
$ echo hi | ./build/grep -i 'H.'      # full pipeline

05 Architecture

Three layers, hand-written.

Entry → dispatcher → applet. Every syscall direct, every buffer in .bss or on the function's stack frame.

06 Why "rill"?

Small. Fast. Cuts through stone.

A rill is a small, fast-running stream — narrow, sharp, cuts through stone over time. Felt fitting for the assembly cousin in the multi-call family: a slimmer, lower-level companion to the Rust jib, Go topsail, and Python mainsail — same shape, different substrate.

Rill is a userland Unix utility binary — no privileged operations, no network code, no exploitation surface. Vulnerabilities go through private security advisories, never public issues.

Adding an applet is mechanical: drop src/applets/<name>.asm, wire it into start.asm's table, add a bats file. The Makefile auto-globs.
